# Скрипт очистки памяти мультиагентной системы

## Описание

`clear_memory.py` - это утилита командной строки для управления памятью мультиагентной системы. Скрипт позволяет просматривать статистику, очищать память по сессиям и агентам.

## Установка и требования

Скрипт использует существующую систему памяти проекта. Перед использованием убедитесь, что:

1. Активирована виртуальная среда проекта
2. Все зависимости установлены
3. Скрипт запускается из корневой директории проекта

```bash
source .venv/bin/activate
```

## Использование

### Быстрый способ (рекомендуется)

Для очистки Text-to-SQL сессии используйте быстрый скрипт:

```bash
# Одной командой очистить память Text-to-SQL сессии
./memory/quick_clear.sh
```

### Основные команды

```bash
# Показать справку
python memory/clear_memory.py --help

# Показать список всех сессий в памяти
python memory/clear_memory.py --list

# Показать статистику конкретной сессии
python memory/clear_memory.py --stats "session_id_here"

# Очистить всю память сессии
python memory/clear_memory.py --clear "session_id_here"
```

### Продвинутые команды

```bash
# Очистить только тактическую память сессии
python memory/clear_memory.py --clear "session_id" --type tactical

# Очистить только стратегическую память сессии  
python memory/clear_memory.py --clear "session_id" --type strategic

# Очистить память конкретного агента
python memory/clear_memory.py --clear "session_id" --agent "schema_rag_agent"

# Очистить конкретный шаг агента
python memory/clear_memory.py --clear "session_id" --agent "nlu_agent" --step 2

# ⚠️ ОПАСНО: Очистить ВСЮ память системы
python memory/clear_memory.py --clear-all
```

### Примеры реального использования

#### Быстрая очистка Text-to-SQL сессии (рекомендуется)

Для удобства создан специальный скрипт `quick_clear.sh`, который автоматически очищает память Text-to-SQL сессии:

```bash
# Быстрая очистка с автоматической статистикой
./memory/quick_clear.sh
```

Этот скрипт автоматически:
- ✅ Активирует виртуальную среду (если нужно)
- ✅ Показывает статистику ДО очистки
- ✅ Очищает память сессии `duckdb_users_kosoj_documents_multiagent_data_sber_index_prod_db`
- ✅ Показывает статистику ПОСЛЕ очистки
- ✅ Подтверждает успешное завершение

#### Ручная очистка проблемной сессии Text-to-SQL

```bash
# 1. Сначала посмотрим статистику
python memory/clear_memory.py --stats "duckdb_users_kosoj_documents_multiagent_data_sber_index_prod_db"

# 2. Очистим всю память сессии
python memory/clear_memory.py --clear "duckdb_users_kosoj_documents_multiagent_data_sber_index_prod_db"

# 3. Проверим, что память очищена
python memory/clear_memory.py --stats "duckdb_users_kosoj_documents_multiagent_data_sber_index_prod_db"
```

#### Очистка конкретного агента

```bash
# Очистить только schema_rag_agent, если он содержит устаревшую информацию
python memory/clear_memory.py --clear "session_id" --agent "schema_rag_agent"
```

## Типы памяти

### Тактическая память (tactical)
- Хранит пошаговые результаты работы агентов
- Используется для межагентной координации
- База данных: SQLite таблица `agent_memory` + ChromaDB коллекция `tactical_memory`

### Стратегическая память (strategic)  
- Хранит цели, контекст и долгосрочную информацию
- База данных: SQLite таблица `strategic_memory` + ChromaDB коллекция `strategic_memory`

## Безопасность

### Меры предосторожности
- Операция `--clear-all` требует подтверждения пользователя
- Все операции очистки необратимы
- Рекомендуется сначала просмотреть статистику с `--stats`

### Рекомендации
1. **Всегда делайте резервную копию** важных данных перед массовой очисткой
2. **Используйте `--list`** для просмотра всех сессий перед очисткой
3. **Тестируйте на отдельной сессии** перед применением к продакшн данным

## Устранение неполадок

### Ошибки импорта
```bash
❌ Ошибка импорта: No module named 'memory.tools'
```
**Решение**: Убедитесь, что скрипт запускается из корневой директории проекта.

### Ошибки подключения к БД
```bash
❌ Ошибка: database is locked
```
**Решение**: Завершите все процессы агентов перед очисткой памяти.

### Пустая статистика
```bash
📭 Сессий не найдено
```
**Это нормально**: Память уже очищена или система новая.

- `memory/database.py` - Обработчик базы данных
