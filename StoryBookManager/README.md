# StoryBook Manager

Десктопное приложение для управления проектами сказок, созданными с помощью storybook_pipeline.

## 🚀 Основные возможности

- **Управление проектами**: Просмотр, создание, удаление и backup проектов сказок
- **Структурированное редактирование JSON**: Удобные формы для редактирования всех типов данных проекта
- **Просмотр медиа**: Встроенный просмотрщик изображений и видео файлов
- **Интеграция с pipeline**: Запуск полного workflow или отдельных этапов генерации
- **Валидация данных**: Проверка согласованности и корректности данных проекта
- **Система backup**: Автоматическое создание резервных копий при изменениях

## 📋 Требования

- Python 3.9+
- tkinter (встроен в Python)
- Доступ к корневой директории проекта MultiAgent

## 🔧 Установка

1. Убедитесь, что вы находитесь в корневой директории проекта MultiAgent:
```bash
cd /path/to/MultiAgent
```

2. Установите зависимости для StoryBook Manager:
```bash
pip install -r StoryBookManager/requirements.txt
```

3. Запустите приложение:
```bash
python StoryBookManager/main.py
```

## 🎯 Основные функции

### Управление проектами
- Просмотр списка всех проектов сказок
- Превью проекта с основной информацией
- Создание backup проектов
- Удаление проектов с автоматическим backup

### Редактирование данных
- **Структурированные формы** для основных типов данных:
  - `00_brief.json` - техническое задание сказки
  - `20_story/story.json` - текст сказки по страницам
  - `20_bible/characters.json` - персонажи
  - `20_bible/locations.json` - локации
  - `97_shots/shots.json` - видеокадры

- **Raw JSON редактор** для продвинутых пользователей
- **Валидация в реальном времени** с подсказками
- **Автоматическое создание backup** при сохранении

### Просмотр медиа
- Просмотр всех изображений проекта:
  - Иллюстрации книги (`50_images/`)
  - Референсы персонажей и локаций (`20_bible/references/`)
  - Кадры для видео (`97_shots/`)
- Превью видео файлов
- Информация о файлах (размер, формат, дата изменения)

### Генерация контента
- **Запуск полного pipeline** для создания сказки
- **Частичная генерация** с выбранного шага
- **Селективная регенерация** изображений и видео
- **Мониторинг прогресса** выполнения в реальном времени
- **Логи выполнения** с возможностью сохранения

## 🔧 Настройки

Настройки приложения сохраняются в `StoryBookManager/config/settings.json`:

```json
{
  "projects_directory": "plots/storybooks",
  "backup_directory": "StoryBookManager/backups", 
  "auto_save_interval": 30,
  "max_backup_files": 10,
  "json_editor_mode": "structured",
  "window_geometry": "1200x800"
}
```

## 📝 Типы редактируемых файлов

### 1. Brief (00_brief.json)
Техническое задание сказки:
- Название и описание
- Целевая аудитория
- Главные персонажи и локации
- Параметры генерации

### 2. Story (20_story/story.json)
Текст сказки:
- Заголовок книги
- Страницы с заголовками и текстом
- Счетчик слов на странице

### 3. Characters (20_bible/characters.json)
Персонажи сказки:
- Основные атрибуты (имя, возраст, роль)
- Неизменные черты (внешность, пропорции)
- Переменные атрибуты (одежда, аксессуары)
- Референсы изображений

### 4. Locations (20_bible/locations.json)
Локации сказки:
- Название и описание
- Ключевые объекты
- Атмосфера и освещение
- Цветовая палитра

### 5. Shots (97_shots/shots.json)
Видеокадры для анимации:
- Параметры сцен и кадров
- Промпты для генерации
- Технические параметры
- Пути к результирующим файлам

## 🔍 Валидация данных

Приложение включает систему валидации:

- **JSON Schema валидация** для проверки структуры данных
- **Согласованность между файлами** (персонажи в brief должны быть в characters)
- **Бизнес-правила** (количество слов на странице, длительность кадров)
- **Предложения по улучшению** контента

## 🔄 Интеграция с Workflow

StoryBook Manager интегрируется с существующим `workflow.enhanced_engine`:

- Использует `EnhancedWorkflowEngine` для запуска pipeline
- Интегрируется с `custom_tools.storybook.*` для селективной генерации
- Поддерживает мониторинг прогресса выполнения
- Сохраняет логи выполнения

## 📊 Структура данных проекта

Каждый проект сказки имеет следующую структуру в `plots/storybooks/{project_id}/`:

```
{project_id}/
├── 00_brief.json              # Техзадание
├── 10_synopsis/               # Синопсис и структура
├── 20_bible/                  # Персонажи, локации, правила
├── 20_story/                  # Текст сказки
├── 30_style/                  # Стиль текста и изображений
├── 40_prompts/                # Промпты для генерации
├── 50_images/                 # Иллюстрации книги
├── 90_md/                     # Markdown версия
├── 91_screenplay/             # Сценарий для видео
├── 95_pdf/                    # PDF книги
└── 97_shots/                  # Видеокадры и анимация
```

## 🚀 Быстрый старт

1. Запустите приложение из корневой директории MultiAgent
2. На вкладке "Проекты" выберите существующий проект (например, "ryaba")
3. Перейдите на вкладку "Редактор" для редактирования JSON файлов
4. Используйте вкладку "Медиа" для просмотра изображений и видео
5. Вкладка "Генерация" позволяет запускать pipeline и регенерировать контент

## 🔧 Разработка

Для разработки и расширения функциональности:

1. Основная логика находится в модулях `core/`
2. GUI компоненты в `gui/`
3. Конфигурация и схемы в `config/`
4. Утилиты в `utils/`

### Добавление нового типа файла для редактирования:

1. Добавьте JSON схему в `config/schemas.py`
2. Расширьте `core/file_manager.py` для работы с новым типом
3. Добавьте форму редактирования в `gui/editor_panel.py`
4. Обновите валидацию в `utils/json_validator.py`

## 📝 Логирование

Логи приложения сохраняются в:
- `StoryBookManager/logs/storybook_manager.log` - основные логи
- `StoryBookManager/logs/storybook_manager_errors.log` - только ошибки
