# MultiAgent System - Streamlit UI

Веб-интерфейс для управления системой мультиагентов на базе Streamlit.

## 🚀 Быстрый старт

### Предварительные требования

1. **Активируйте виртуальное окружение:**
   ```bash
   source .venv/bin/activate
   ```

2. **Установите дополнительные зависимости для Streamlit:**
   ```bash
   pip install streamlit plotly pandas
   ```

### Запуск приложения

```bash
# Из корневой директории проекта
streamlit run streamlit_app/app.py

# Или из директории streamlit_app
cd streamlit_app
streamlit run app.py
```

Приложение будет доступно по адресу: http://localhost:8501

## 🔧 Технические детали

### Архитектура

Streamlit приложение использует **тонкий API слой** для взаимодействия с основной системой:

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Streamlit UI  │ -> │   Public APIs   │ -> │   Core System   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### API модули

- `workflow/streamlit_api.py` - API для управления workflows
- `agent_streamlit_api.py` - API для управления агентами
- `db_plugins/streamlit_api.py` - API для плагинов БД
- `text_to_sql_streamlit_api.py` - API для Text-to-SQL
- `memory/streamlit_api.py` - API для системы памяти
- `configuration_api.py` - API для конфигурации
- `telemetry/smolagents_telemetry.py` - Телеметрия
- `unified_logging.py` - Унифицированное логирование

### Особенности интеграции

1. **Проверка виртуального окружения** - автоматическая проверка при запуске
2. **Динамическое обнаружение** - автоматическое сканирование профилей и пайплайнов
3. **Локальная телеметрия** - сохранение трасс в JSONL файлы
4. **Соблюдение правил проекта** - DSN с схемой, лимиты через API плагинов
5. **Обратная совместимость** - без изменений в основном коде

### Состояние сессии

Streamlit использует `st.session_state` для хранения:
- Выбранных пайплайнов и агентов
- Результатов выполнения
- Истории SQL запросов
- Настроек фильтров
- Сохраненных конфигураций

## 🔒 Безопасность

### Встроенная защита

1. **Валидация DSN** - обязательная схема в строке подключения
2. **SQL безопасность** - валидация через плагины БД
3. **Лимиты выполнения** - применение через API, не через SQL
4. **Логирование действий** - все операции записываются

### Рекомендации

- Используйте только SELECT операции для Text-to-SQL
- Регулярно проверяйте логи безопасности
- Настройте ограничения доступа к таблицам
- Включите детальную телеметрию

## 🐛 Устранение неполадок

### Частые проблемы

1. **"Virtual environment не активировано"**
   ```bash
   source .venv/bin/activate
   ```

2. **Ошибки импорта модулей**
   - Убедитесь, что запускаете из корневой директории проекта
   - Проверьте наличие `__init__.py` файлов

3. **Порт уже используется**
   ```bash
   streamlit run app.py --server.port 8502
   ```

4. **Медленная загрузка страниц**
   - Проверьте настройки телеметрии
   - Отключите автообновление на тяжелых страницах

### Логи и отладка

1. **Логи Streamlit** - выводятся в консоль при запуске
2. **Логи системы** - доступны на странице Logs/Traces
3. **Трассы выполнения** - сохраняются в `logs/traces/`
4. **Конфигурация** - экспорт/импорт на странице Settings
